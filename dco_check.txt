4273315c Document tool result size limits and filesystem storage (#1805)
## Summary
This PR adds comprehensive documentation for HolmesGPT's tool result
size limiting and filesystem storage features, explaining how the system
handles large tool responses that exceed the LLM context window.

## Key Changes
- **New environment variables section** in
`docs/reference/environment-variables.md`:
- `HOLMES_TOOL_RESULT_STORAGE_ENABLED`: Controls whether large results
are saved to disk (default: `true`)
- `HOLMES_TOOL_RESULT_STORAGE_PATH`: Configurable directory for storing
large results (default: `/tmp/.holmes`)
- `TOOL_MAX_ALLOCATED_CONTEXT_WINDOW_PCT`: Percentage-based limit for
tool responses (default: `15%`)
- `TOOL_MAX_ALLOCATED_CONTEXT_WINDOW_TOKENS`: Absolute token limit for
tool responses (default: `25000`)

- **Updated bash toolset documentation** in
`docs/data-sources/builtin-toolsets/bash.md`:
  - Explains how large tool results are automatically saved to disk
- Documents that bash toolset allows read-only commands (`cat`, `head`,
`tail`, `wc`, `jq`) on the storage directory
- Added reference table for the two most relevant storage configuration
variables
  - Cross-references the full environment variables documentation

## Notable Details
- The documentation clarifies the fallback behavior: when
`HOLMES_TOOL_RESULT_STORAGE_ENABLED` is disabled, oversized results are
dropped and the LLM is asked to retry with a narrower query
- Explains that the effective limit is the **minimum** of the
percentage-based and absolute token limits
- Includes practical examples for each configuration option
- Emphasizes that storage directories are automatically cleaned up per
session
